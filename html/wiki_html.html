表单

<form>

定义供用户输入的 HTML 表单

<form method="传送方式" action="服务器文件">
GET（默认方法）：
如果表单提交是被动的（比如搜索引擎查询），并且没有敏感信息。
当您使用 GET 时，表单数据在页面地址栏中是可见的：
action_page.php?firstname=Mickey&lastname=Mouse
注释：GET 最适合少量数据的提交。浏览器会设定容量限制。
POST：
如果表单正在更新数据，或者包含敏感信息（例如密码）。
POST 的安全性更加，因为在页面地址栏中被提交的数据是不可见的。

autocomplete属性：规定表单或输入字段是否应该自动完成。
当自动完成开启，浏览器会基于用户之前的输入值自动填写值。autocomplete="yes/on"
novalidate属性：规定在提交表单时不对表单数据进行验证。
form属性：规定元素所属的一个或多个表单。

<form action="action_page.php" id="form1">
...
</form>
<input type="text" name="lname" form="form1" />
<input>

定义输入控件

<input type="text/password" name="名称" value="文本" />

name：为文本框命名，以备后台程序使用
value：为文本输入框设置默认值(一般起到提示作用)
<input type="radio/checkbox" value="值" name="名称" checked="checked" />
malefemale
value：提交数据到服务器的值
name：为控件命名，以备后台程序使用 注意:同一组的单选按钮，name 取值一定要一致
<input type="submit" value="提交" />
<input type="reset" value="重置" />
<input type="hidden" />隐藏

HTML5 输入类型：
number date color range month week time datetime datetime-local email search tel url
<input type="number" name="points" min="0" max="100" step="10" value="30" /> 用于应该包含数字值的输入字段。
<input type="date" name="bday" max="1979-12-31" min="2000-01-02" /> 用于应该包含日期的输入字段。
<input type="color" name="favcolor" value="#ff0000" /> 用于应该包含颜色的输入字段。
<input type="range" name="points" min="0" max="10" /> 用于应该包含一定范围内的值的输入字段。
<input type="month" name="bdaymonth" /> 允许用户选择月份和年份。
<input type="week" name="year_week" />允许用户选择周和年
input type="time" name="usr_time" />允许用户选择时间（无时区）。
<input type="datetime" name="bdaytime" /> 允许用户选择日期和时间（有时区）。
<input type="datetime-local" name="bdaytime" />允许用户选择日期和时间（无时区）。

<input type="email" name="email" />用于应该包含电子邮件地址的输入字段。根据浏览器支持，能够在被提交时自动对电子邮件地址进行验证。
<input type="search" />用于搜索字段（搜索字段的表现类似常规文本字段）。
<input type="tel" />用于应该包含电话号码的输入字段。
<input type="url" />用于应该包含 URL 地址的输入字段。根据浏览器支持，在提交时能够自动验证 url 字段。
<input type="file" name="img" multiple />用于提交文件

value属性：规定输入字段的初始值。
readonly属性：规定输入字段为只读（不能修改）。readonly属性不需要值。它等同于readonly="readonly"。
disabled属性：规定输入字段是禁用的。被禁用的元素是不可用，不可点击，不会被提交。disabled 属性不需要值。它等同于 disabled="disabled"。
required属性规定文本区域是必填的。required="required"
size属性：规定输入字段的尺寸（以字符计）。
maxlength属性：规定输入字段允许的最大长度。
height width属性：规定元素的高度和宽度。
list属性：引用的<datalist>元素中包含了<input>元素的预定义选项。
min max属性：规定<input>元素的最小值和最大值。
multiple属性：<multiple>属性是布尔属性。如果设置，则规定允许用户在<input> 元素中输入一个以上的值。
step属性：规定<input>元素的合法数字间隔。示例：如果 step="3"，则合法数字应该是 -3、0、3、6、等等。

<section> IE9+ Firefox Chrome Safari Opera

定义 section。表示页面中的一个内容区块，比如章节、页眉、页脚或页面的其他部分。可以和h1、h2...等标签结合起来使用，表示文档结构

   <h1>WWF</h1>
   <p>The World Wide Fund for Nature (WWF) is....</p>
</section>

HTML 全局属性

accesskey

规定激活元素的快捷键。

除了 Opera,所有浏览器均支持。
定义和用法： accesskey 属性规定激活（使元素获得焦点）元素的快捷键。
注释：以下元素支持 accesskey 属性：a, area, button, input, label, legend 以及 textarea。
语法：<element accesskey="character">

class

规定元素的一个或多个类名（引用样式表中的类）。

注释：class 属性不能在以下 HTML 元素中使用：base, head, html, meta, param, script, style 以及 title。
提示：可以给 HTML 元素赋予多个 class，例如：<span class="left_menu important">。这么做可以把若干个 CSS 类合并到一个 HTML 元素。
提示：类名不能以数字开头！只有 Internet Explorer 支持这种做法。

contenteditable IE Firefox Chrome Safari Opera

规定元素内容是否可编辑。

<element contenteditable="true|false">

contextmenu Firefox

规定元素的上下文菜单。上下文菜单在用户点击元素时显示。

目前只有 Firefox 支持 contextmenu 属性

data-* IE Firefox Chrome Safari Opera

用于存储页面或应用程序的私有定制数据。

定义和用法：
data-* 属性用于存储页面或应用程序的私有自定义数据。
data-* 属性赋予我们在所有 HTML 元素上嵌入自定义 data 属性的能力。
存储的（自定义）数据能够被页面的 JavaScript 中利用，以创建更好的用户体验（不进行 Ajax 调用或服务器端数据库查询）。
data-* 属性包括两部分：
1.属性名不应该包含任何大写字母，并且在前缀 "data-" 之后必须有至少一个字符
2.属性值可以是任意字符串
注释：用户代理会完全忽略前缀为 "data-" 的自定义属性。
语法：<element data-*="somevalue">

dir

规定元素中内容的文本方向。

<element dir="ltr|rtl">

draggable IE9+ Firefox Chrome Safari Opera

规定元素是否可拖动。

<element draggable="true|false|auto">

dropzone none

规定在拖动被拖动数据时是否进行复制、移动或链接。

目前所有主流浏览器都不支持 dropzone 属性。

hidden IE Firefox Chrome Safari Opera

规定元素仍未或不再相关。

<element hidden="hidden">

id

规定元素的唯一 id。

id 属性规定 HTML 元素的唯一的 id。
id 在 HTML 文档中必须是唯一的。
id 属性可用作链接锚（link anchor），通过 JavaScript（HTML DOM）或通过 CSS 为带有指定 id 的元素改变或添加样式。

lang

规定元素内容的语言。

<element lang="language_code">

spellcheck IE10+ Firefox Chrome Safari Opera

规定是否对元素进行拼写和语法检查。

定义和用法：
spellcheck 属性规定是否对元素进行拼写和语法检查。
可以对以下内容进行拼写检查：
input 元素中的文本值（非密码）
<textarea> 元素中的文本
可编辑元素中的文本
语法：<element spellcheck="true|false">

style

规定元素的行内 CSS 样式。

style 属性规定元素的行内样式
style="color:blue; text-align:center"

tabindex

规定元素的 tab 键次序。

除了 Safari，所有浏览器均支持。
tabindex 属性规定元素的 tab 键控制次序（当 tab 键用于导航时）。

title

规定有关元素的额外信息。

title 属性规定关于元素的额外信息。
这些信息通常会在鼠标移到元素上时显示一段工具提示文本（tooltip text）。
提示：title 属性常与 form 以及 a 元素一同使用，以提供关于输入格式和链接目标的信息。同时它也是 abbr 和 acronym 元素的必需属性。

translate none

规定是否应该翻译元素内容。

所有主流浏览器都无法正确地支持 translate 属性。
